# Chameleon Asset Studio ユーザーガイド

最終更新日: 2026-07-16
対象バージョン: 0.1.0

このガイドは、Chameleon Asset Studio でゲーム用 2D アセットを作って書き出すまでの使い方を説明します。仕様の正本は `docs/REQUIREMENTS_SPECIFICATION.md`、データ形式は `docs/DATA_FORMAT.md` を参照してください。

---

## 1. はじめる

1. ホーム画面で「プロジェクト名」を入力し「作成」を押すと、編集画面が開きます。
2. 既存の `.casproj` ファイルがある場合は「.casproj を読み込む」から取り込めます（別プロジェクトとして追加されます）。
3. 作業は自動保存されます。画面上部の「保存済み」表示が目印です。データはブラウザ内（IndexedDB）に保存されます。

## 2. 画像を取り込む

- 編集画面へ PNG / JPG / WebP をドラッグ&ドロップするか、「画像を選ぶ」から選択します。
- 最大サイズは 4096 x 4096 です。超えると理由が表示されます。
- 元画像は破壊的編集をせずに保持されます（編集用・サムネイルと分けて保存）。

### 2.1 画像を取り込まずに新規アセットを作る（2D-2-CREATE-01、2026-07）

プロパティパネルの「アセット」欄にある「新規アセットを作成」フォームで、画像を取り込まなくても空キャンバスのアセットを作れます。

- 「新規アセット名」「新規アセットの種別」（既存の全アセット種別から選べます）「新規アセットのサイズ」（32 / 64 / 128 / 256 の正方形プリセット、既定 64）を指定して「新規アセットを作成」を押します。
- 中身は透明な画像です（取り込み画像と同じ source / edit / thumbnail の 3 テクスチャ構成）。レイヤーは 1 枚（透明画像、位置は (0, 0)）で、原点は既存の既定どおり足元中央になります。
- **character** を選んだときだけ、当たり判定「body」（矩形）が最初から付きます。他の種別は空キャンバスのみです。この対応表はコードで管理しており、`.casproj` / asset.json には「テンプレートを適用した結果」（当たり判定そのもの）だけが通常のフィールドとして保存されます。テンプレートという概念自体は保存データには現れません。
- 作成したアセットはすぐに選択状態になり保存されます。プロジェクト単位の操作のため、**Undo / Redo の対象外**です。

### 2.2 アセットを削除する（2D-2-CREATE-01、2026-07）

プロパティパネルの「アセット」欄の「アセットを削除」で、選択中のアセットを削除できます。

- 確認ダイアログで防護されます。削除すると画像 Blob・復旧点も含めてまとめて消え、元に戻せません。
- 最後の 1 件を削除してもアセット 0 件の状態になり、「アセットがありません。画像を取り込むか、新規アセットを作成してください。」という空状態が表示されます（0 件でも画面は壊れません）。
- 確認ダイアログで誤操作を防ぐ設計のため、**Undo / Redo の対象外**です。

### 2.3 選択中のアセットを独立コピーする（2D-2-PROJECT、2026-07）

プロパティパネルの「独立コピーを作成」で、選択中のアセットを同じProject内へ複製できます。

- Asset、TextureRef、レイヤー、パーツ、フレーム、アニメーションなどは新しいIDへ置き換えられ、画像Blobも新しいAsset用に複製されます。
- コピー後は元と別のアセットです。片方の編集・削除はもう片方へ反映されません。
- 現在はFamily / Variantやlinked variantではありません。自動追従や元への逆反映は行いません。
- Project単位の追加操作のため、**Undo / Redoの対象外**です。
- アセット一覧には種別とキャンバスサイズが表示されます。種別変更はAsset本体とProject内の要約へまとめて保存されます。

## 3. 編集する

- **キャンバス**: ズーム / パン、レイヤーの選択・ドラッグ移動・数値入力。Undo / Redo（Ctrl+Z / Ctrl+Y）対応。
- **グリッド / スナップ**: キャンバス下部でグリッド表示とスナップを切り替えられます。grid size は 8px / 16px / 32px / custom から選べます。スナップ ON のとき、レイヤー、原点、アンカー、矩形 / 円の当たり判定の座標をドラッグまたは数値入力で更新すると、座標が grid size に丸められます。既存データを開いただけでは座標は変更されず、保存データの座標単位は引き続き px です。
- **画像編集ツール**: トリミング、単色背景の透過、消しゴム、色相 / 彩度 / 明度、パレット置換、輪郭線。重い処理は Web Worker で実行され、進捗が表示されます。
- **レイヤー**: 追加（画像 / ガイド）、名前変更、表示 / ロック、並べ替え、削除。
- **パーツ**: レイヤーにチェックを付けて「パーツを作成」。種別（head / body / …）、pivot、親パーツ、バインドポーズ、可動域を設定できます。

## 4. ゲーム用データを付ける

- **原点**: 原点ツールまたは数値入力で設定。キャラクターは足元中央が基本です。
- **アンカー**: 用途（foot / hand_left / weapon など）付きの参照座標。
- **当たり判定**: 矩形 / 円。用途（body / attack / pickup など）と表示切替。ツールバーの「判定」ツールでキャンバス上から直接、選択・移動・リサイズもできます（詳細は 8 章「Phase 19-C 判定表示・選択」を参照）。
- **ゲーム属性**: アイテムの score / rarity など自由な key-value（数値文字列は自動で数値になります）。
- **種別ごとの設定**: 背景レイヤー（役割 / パララックス速度 / ループ + プレビュー）、タイル（tileSize / 衝突タイプ）、ギミック（movementPreset）、アイテムテンプレート。

## 5. アニメーションを作る

- **フレーム**: 現在の表示状態をフレームとして保存し、並べ替え・複製・削除。
- **アニメーション**: idle / walk などを作成し、fps / ループを設定して再生確認。
- **簡易リグ**: パーツの親子関係とキーフレーム（位置 / 回転 / 拡大）を編集し、「フレームへ焼き込み」で通常のフレームアニメーションに変換します。モーションテンプレート（idle_sway / walk_bounce / jump_squash / attack_swing / damage_shake / dead_collapse）を適用してから手動調整もできます。

## 6. 書き出す

書き出しパネルから次をダウンロードできます。

| 種類 | 用途 |
| --- | --- |
| PNG / WebP | 合成済み 1 枚画像 |
| asset.json | ゲーム用メタデータ（原点・アンカー・判定・アニメ込み） |
| ZIP | 画像 + atlas + サンプル HTML + helper + エンジンガイド一式 |
| .casproj | プロジェクトの保存ファイル（画像込み。バックアップ・受け渡し用） |

ZIP の中身と使い方は `docs/EXPORT_FORMATS.md`、Godot / Unity / Rive / Spine との関係は `docs/ENGINE_INTEGRATION.md` を参照してください。ZIP 内の `examples/` は `npx serve .` などのローカルサーバーで開きます。

## 7. スマホ / iPad で使う

- スマホでは下部ナビで「編集 / プロパティ / タイムライン / 書き出し」を切り替えます。軽い編集・確認・書き出しを想定しています。
- iPad / PC では全パネルを同時に表示でき、主要編集がすべて行えます。

## 8. 困ったとき

| 症状 | 原因と対処 |
| --- | --- |
| 「画像サイズが大きすぎます」 | 4096 x 4096 以下に縮小してから取り込んでください |
| 「この環境では WebP 書き出しに対応していません」 | ブラウザが WebP エンコード非対応です。PNG を使ってください（ZIP は WebP だけ省いて生成されます） |
| 「画像 Blob が見つかりません」 | 参照画像がブラウザ保存領域から失われています。該当レイヤーの画像を取り込み直してください（画像欠けのままの書き出しは安全のため停止します） |
| サンプル HTML で画像が出ない | `file://` では動きません。フォルダをローカルサーバーで開いてください |
| ブラウザのデータ消去で作品が消えた | IndexedDB 保存のため消去で失われます。重要な作品は定期的に `.casproj` を書き出して保管してください |
| プロジェクトを誤って削除した | ホーム画面の「ごみ箱」から復元できます（8.1 参照） |
| 画像編集で崩れた見た目を戻したい | プロパティの「復旧点」から編集前の状態に戻せます（8.2 参照） |
| 保存に失敗する（容量不足） | 「保存容量が不足しています」と表示されたら、ごみ箱を空にするか不要なプロジェクトを削除して空き容量を確保してください（8.3 参照） |
| 読み込んだ `.casproj` が壊れていた | ホーム画面の「読み込みに失敗したファイル」に理由付きで残ります（8.4 参照） |

### 8.1 ごみ箱（2D-1B-STORAGE、2026-07）

プロジェクトの「削除」はごみ箱への移動になり、即座には失われません。

- ホーム画面の「ごみ箱」セクションに、削除したプロジェクトの名前・削除日時が表示されます。
- 「復元」で元の一覧へ戻せます（アセット・画像も復元されます）。
- 「完全に削除」（確認あり）または「ごみ箱を空にする」で、二度と復元できない完全削除になります。
- ごみ箱には最大 5 件まで残ります。6 件目を削除すると、最も古いプロジェクトが自動的に完全削除されます。
- ごみ箱にある間は画像 Blob と、そのプロジェクトの復旧点（8.2）も保持されます。完全削除時にまとめて消えます。

### 8.2 復旧点（破壊的画像編集からの復旧、2026-07）

トリミング・消しゴム・色調整・パレット置換・輪郭線などの画像編集は、適用前の状態を「復旧点」として自動保存します。

- プロパティパネルの「復旧点」に、操作名（例: 「消しゴム」）と時刻が新しい順に並びます。
- 「復元」を押すと画像とアセットの状態が復旧点の内容に戻ります。復元も通常の操作と同じく Undo / Redo（Ctrl+Z / Ctrl+Y）できます。
- アセットあたり最大 3 件まで保持し、超えると最も古い復旧点から自動的に削除されます。
- 復旧点はこのブラウザの保存領域（IndexedDB）だけに残る一時的なデータで、`.casproj` の書き出しには含まれません。

### 8.3 容量不足のとき

ブラウザの保存容量が足りずに保存へ失敗すると、画面上部に「保存容量が不足しています。ごみ箱を空にするか、不要なプロジェクトを削除して空き容量を確保してください。」と表示されます。ごみ箱（8.1）を空にする、または使わなくなったプロジェクトを完全に削除してから、もう一度操作をやり直してください。

### 8.4 読み込みに失敗したファイル（2026-07）

壊れている、または `.casproj` として読み込めないファイルは、正本データへは一切書き込まれません。ホーム画面の「読み込みに失敗したファイル」に、ファイル名・日時・失敗理由が表示されます（最新 3 件まで。50MB を超えるファイルは理由とサイズのみ保持し、元データは保存しません）。原因調査が終わったら「削除」で一覧から消せます。

### Phase 19-C 判定表示・選択（2026-07）

当たり判定パネルには用途色の凡例が表示されます。body / attack / pickup / sensor / custom は用途名と色で確認でき、sensor はキャンバス上の破線と凡例・一覧の縞スワッチでも通常判定と区別できます。

当たり判定一覧の各行には用途色スワッチと「選択」ボタンがあります。選択した判定行は強調表示され、キャンバス上の同じ判定も太線と halo で強調されます。選択状態は編集補助の UI 状態であり、asset.json などの保存データには含まれません。

### Phase 19-C 判定ツール（キャンバス直接編集、2026-07）

ツールバーの「判定」を選ぶと、キャンバス上で当たり判定を直接編集できます。

- **選択**: 判定の本体（矩形の内側 / 円の内側）をクリックすると選択します。一覧の「選択」ボタンとキャンバス上のクリックは相互に連動します。判定を表示していない（「判定を表示」OFF）ときはキャンバス操作を受け付けません。
- **移動**: 選択した判定の本体をドラッグすると移動します。サイズは変わりません。
- **リサイズ（矩形）**: 選択した矩形判定には四隅に白いハンドルが表示されます。ハンドルをドラッグすると、つかんだ角の対角は固定したままサイズを変えられます。最小サイズは 1px で、対角を越えて反転することはありません。
- **半径変更（円）**: 選択した円判定には中心から右へ伸びた位置にハンドルが表示されます。ドラッグすると中心を変えずに半径だけ変わります。最小半径は 1px です。
- **スナップ**: キャンバス下部の「スナップ」が ON のとき、移動・リサイズ後の座標はグリッドサイズに丸められます。OFF のときは整数へ丸められます。
- **Undo / Redo**: 移動・リサイズはそれぞれ「判定移動」「判定リサイズ」「判定半径変更」として履歴に積まれ、Ctrl+Z / Ctrl+Y または画面上部のボタンで戻せます。
