## 0. 完成系での位置づけ（逆算リンク）

由来する完成系（`features/README.md` 2 章）:

- **D-DONE-8**: プロジェクトを保存し、開き直して**完全に再開**でき、クラッシュしても復旧できる。
- **T-DONE-9**: 保存・再開でき、**元の 3D モデルを絶対に上書きしない**。
- **S-DONE-1**: Home / Project Dashboard から 2D・3D いずれのプロジェクトへも入れる。
- **S-DONE-5**: どのモジュール構成でも同じプロジェクトを開け、未知データを壊さない。

逆算の理由: 上の「done」がすべて成り立つには、**プロジェクトという入れ物を、壊さず・再現性をもって、作り／開き／保存し／再開できる**土台が最初に要る。他のあらゆる機能はこの上で動く。ゆえに詳細化の第 1 機能。

関係マップのノード: `A2 Core 契約` / `D1 保存形式` / `D2 migration` / `D3 構成非依存の不変条件` / `G5 非破壊` / `A9 Extension Host`。

---

## 1. ユーザーストーリー

- 制作者として、**新しいプロジェクトを 2D か 3D を選んで作り**、すぐ作業に入りたい。迷いたくない。
- 制作者として、**保存して閉じ、後日開き直したときに、閉じたときの続きから**再開したい。並び順・選択・付けたデータが失われていては困る。
- 制作者として、**作業中にアプリやマシンが落ちても、直前の状態に戻せる**安心が欲しい。
- 制作者として、**あるモジュールを入れずに開いても、他環境で付けたデータが消えない**ことを信頼したい。
- 制作者として、**元の素材（取り込んだ画像・3D モデル）が勝手に上書きされない**ことを保証してほしい。

---

## 2. 用語

- **Project**: 1 つの制作単位を表す入れ物（メタ情報 + Document + 参照素材 + 付与データ）。保存の対象。
- **Document**: Project 内の編集対象データ本体（2D は canvas/layer/frame 等、3D は scene/anchor/collider 等。中身はエディション別で、本機能は器を定義）。
- **UI 一時状態**: ズーム・選択・パネル開閉など、**保存データに混ぜない**揮発状態（再開時の復元は 6 章で扱う）。

---

## 3. 機能要件（FR）

### 3.1 作成

- **FR-1** 新規プロジェクトを作成できる。作成時に **エディション（2D / 3D）** を選ぶ。エディションは作成後に混在・変更しない（別プロジェクトにする）。
- **FR-2** 作成の起点を選べる（2D: 空 / テンプレート / 画像取り込み / 図形 / パーツ、3D: ファイル取り込み）。起点の実体は各作成機能（F-2D-01 / F-3D-01）が担い、本機能は「空の Project を用意し、起点機能へ渡す」ところまでを負う。
- **FR-3** プロジェクト名・作成日時・エディション・スキーマ版などの**メタ情報**を持つ。

### 3.2 開く

- **FR-4** 既存プロジェクトを開ける。開いた直後、保存時の Document 状態が再現される。
- **FR-5** 最近使ったプロジェクトの一覧から開ける（Dashboard 連携: F-CORE-08）。
- **FR-6** 開く対象のスキーマ版が現行より古い場合、**migration（F-CORE-05）** を通してから開く。新しすぎて解釈できない場合は、**壊さず**に「未知データとして保持」し、その旨を提示する（S-DONE-5 / D3）。

### 3.3 保存

- **FR-7** 保存できる（上書き保存 / 名前を付けて保存）。保存は Document + 付与データ + メタ + 未知データを 1 つの入れ物にまとめる。
- **FR-8** 保存は **原子的（atomic）** に行う。保存の途中で中断しても、**既存の保存済みファイルを破壊しない**（一時ファイルへ書き切ってから入れ替える等の方式。方式は実装時に確定）。
- **FR-9** 取り込んだ元素材（画像・3D モデル）を**上書きしない**。編集・軽量化は派生として持ち、元は保持する（G5 / 非破壊。詳細は F-CORE-03）。
- **FR-10** 未保存の変更がある状態で閉じる／別プロジェクトを開くとき、**保存を促す**（破棄・保存・キャンセルを選べる）。

### 3.4 再開・セッション

- **FR-11** 開き直したとき、Document の内容（構造・並び・付与データ）が**保存時と一致**する。
- **FR-12** 直近に開いていたプロジェクトを記録し、次回起動時に素早く再開できる（自動で開くかは設定。既定は Dashboard 表示）。
- **FR-13** クラッシュ後の再起動時、**未保存だった直近の一貫状態**を復旧候補として提示する（実体は F-CORE-02 自動保存 & 復旧が担い、本機能は復旧されたセッションを「開く」経路に接続する）。

### 3.5 閉じる

- **FR-14** プロジェクトを閉じられる。閉じる際に UI 一時状態を破棄し、保存対象データと混ざっていないことを保証する。

## 4. 非機能要件（NFR）

- **NFR-1（データ非損失）**: いかなる正常操作でも、保存済みデータを失わない。保存中断・電源断でも、少なくとも直前の保存済み状態は残る（FR-8）。
- **NFR-2（再現性）**: 同一構成で開き直した Document は、保存時とバイト意味上等価（表示順・ID・付与データが一致）。
- **NFR-3（構成非依存）**: モジュール構成が異なっても同じ Project を開け、未知・未対応データは破壊せず保持する（D3）。
- **NFR-4（性能）**: 代表サイズのプロジェクトで、開く／保存が UI を固めない（重い I/O は非同期・別スレッド。F-PLT-03）。開き直しからの再開は体感即時を狙う（具体値は F-PLT-04 で予算化）。
- **NFR-5（オフライン）**: 作成・開く・保存・再開はすべてローカルで完結し、ネットワークを要求しない。
- **NFR-6（非破壊の絶対性）**: 元素材の上書き禁止は例外を作らない（T-DONE-9）。

---

## 5. データ・保存への影響

- 本機能は **保存の「器」** を定義する起点であり、器の**形式・版・migration の正本は F-CORE-05**（関係マップ D1 / D2）。本機能は「器に何を入れ、いつ atomic に書き、どう開くか」のライフサイクル契約を持つ。
- **UI 一時状態を保存データに入れない**（Arch 5 章の規律）。再開時の視点復元が要る場合は、Document とは**別レイヤー**の「セッション状態」として持ち、保存本体と分離する（6 章）。
- **未知データ保持**: 開いた Project に、現在のモジュールが解釈できないフィールドがあっても保持し、保存時に書き戻す（D3 / D4 名前空間拡張）。
- 保存の入れ物が現行 `.casproj` / `asset.json` を**継承するか新設するか**は未決（`APP_ROADMAP_DECISIONS_OPEN_ITEMS.md` APP-O-01）。本機能はどちらでも成り立つ要件として書く。

## 6. UI 一時状態と再開の境界（設計上の要注意点）

「完全に再開」（D-DONE-8）と「UI 一時状態を保存に混ぜない」（規律）は両立させる必要がある。分離方針:

- **保存本体（Document）**: 構造・並び・ID・付与データ。再現性の対象（NFR-2）。
- **セッション状態（任意・分離）**: 最後のズーム・選択・開いていたパネル・スクロール位置。利便のために復元してよいが、**保存本体とは別に持ち**、欠落しても Document は完全に開ける。
- 原則: セッション状態が壊れても Document は無傷で開ける。セッション状態を Document の一部として保存しない。

## 7. 依存機能

- **F-CORE-05**（保存形式・migration）: 器の形式・版・移行の正本。FR-6 / FR-7 の前提。
- **F-CORE-03**（非破壊編集モデル）: FR-9 / NFR-6 の実体。
- **F-CORE-02**（自動保存 & 復旧）: FR-13 の実体。
- **F-CORE-09**（Extension Host）: 開くとき、付与データを持つモジュールの有無を知る経路。未インストールでも FR-6 の未知データ保持で成立。
- **F-CORE-08**（Dashboard）: FR-5 / FR-12 の入口。
- **F-PLT-02 / F-PLT-03**（ネイティブ I/O・offload）: atomic 保存・非同期 I/O の土台。

## 8. 受け入れ条件（Acceptance）

- **AC-1**: 新規作成（2D / 3D 選択）→ 編集 → 保存 → 終了 → 開き直し で、Document が保存時と一致する（NFR-2）。
- **AC-2**: 保存の途中で強制終了させても、直前の保存済みファイルが壊れず開ける（FR-8 / NFR-1）。
- **AC-3**: あるモジュールが付けたデータを含む Project を、そのモジュール未インストールの構成で開いても、**エラーにならず・データが消えず**、保存で書き戻される（NFR-3 / D3）。
- **AC-4**: 取り込んだ元画像 / 元 3D モデルのファイルが、いかなる編集・軽量化・保存でも上書きされていない（NFR-6 / T-DONE-9）。
- **AC-5**: 未保存の変更があるまま閉じようとすると保存が促され、破棄を選ばない限り失われない（FR-10）。
- **AC-6**: セッション状態（ズーム等）が失われても、Document は完全に開ける（6 章）。

## 9. 非目標

- 具体的な保存形式のスキーマ確定（F-CORE-05 の担当）。
- クラウド保存・同期・版管理（本コンセプトの非目標）。
- 複数人同時編集。
- Document の中身（2D の layer 構造、3D の scene 構造）の定義（各エディション機能の担当）。
