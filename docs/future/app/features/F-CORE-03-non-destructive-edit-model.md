## 0. 完成系での位置づけ（逆算リンク）

由来する完成系（`features/README.md` 2 章）:

- **D-DONE-2**: レイヤー・パーツ・フレームを整理し、ラスタ／ベクタを**非破壊で編集・修正**できる。
- **T-DONE-6**: **元を壊さず**軽量化した別モデルを作り、前後を比較して選べる。
- **T-DONE-9**: 保存・再開でき、**元の 3D モデルを絶対に上書きしない**。

逆算の理由: 上記が成り立つには、**編集が元（source）を書き換えず、常に派生（derivation）として積まれ、いつでも元へ戻せて・前後を比較できる**モデルが土台に要る。F-CORE-01（ライフサイクル）と F-CORE-05（保存形式）は「元を上書きしない」を要件として掲げるが、その**実体となる編集モデル**を定義するのが本機能である。

関係マップのノード: `G5 非破壊・元データ保持`（本機能が実現）、`D4 名前空間拡張` / `A2 Core 契約`（保持先）、`F-CORE-05`（永続化）。

## 1. ユーザーストーリー

- 制作者として、**取り込んだ元画像・元モデルを、編集で汚したくない**。いつでも元に戻せる安心が欲しい。
- 制作者として、**軽量化やトリミングをしても、元を残したまま**、後で違う設定でやり直したい。
- 制作者として、**編集前と編集後（軽量化前後）を並べて比較**して選びたい。
- 制作者として、**1 つの元から複数のバリアント**（別解像度・別トリム）を、互いに影響させず作りたい。

## 2. 用語

- **Source（元データ）**: 取り込み／作成時に確定する不変の原本。編集で書き換えない。
- **Derivation（派生）**: Source に対する非破壊の変換・編集の表現。可視結果を生む。
- **Rendered result（結果）**: Source + Derivation を適用した見た目・書き出し対象。
- **Bake / Flatten（焼き込み）**: 派生を確定した新しい派生成果物にすること。**Source は残す**。
- **Variant（バリアント）**: 1 つの Source から派生した、名前付きの複数派生（系譜: ADR-0003）。

## 3. 機能要件（FR）

### 3.1 元データの不変性

- **FR-1** 取り込み／作成した各アセットは、**不変の Source を保持**する。いかなる編集操作も Source を書き換えない。
- **FR-2** 取り込んだ**元ファイルをディスク上で上書きしない**（バイト保持）。F-CORE-01 FR-9 / F-CORE-05 FR-13 の実体。
- **FR-3** Source は保存の器に **一度だけ**持ち、派生は Source を**参照**する（`D4`／F-CORE-05 の名前空間・領域分離に乗る。二重保存を避ける）。

### 3.2 非破壊の派生

- **FR-4** 編集は **Source を書き換えず Derivation に積む**（操作列 or 派生成果物 + 参照）。結果は Source + Derivation の適用で得る。
- **FR-5** **元へ戻す（reset to source）** をいつでも実行できる。
- **FR-6** **Bake / Flatten は破壊的にしない**。焼き込みは新しい派生成果物を作り、**Source を残して再編集可能**にする（系譜: ADR-0004 の「出力は派生」）。
- **FR-7** **1 つの Source から複数 Variant** を作れる。各 Variant は独立し、**互いや Source に影響しない**（系譜: ADR-0003）。

### 3.3 比較と可視性

- **FR-8** **前後比較**を提供する。2D は Source と現派生、3D は軽量化の前後（pre/post optimize）を並べて確認できる（D-DONE-2 の修正確認 / T-DONE-6 の前後選択）。
- **FR-9** どこまでが**元でどこからが編集**かを、利用者が判別できる（可視性。S-DONE-4）。

### 3.4 書き出し・境界

- **FR-10** 書き出し対象は **Rendered result（派生結果）**。Source はプロジェクトに**保持**し、必ずしも書き出しに含めない（再編集のため残す）。
- **FR-11** データ上の境界を明確にする: **Source 領域 / Derivation 領域 / UI 一時状態（保存しない）**。F-CORE-01 6 章・F-CORE-05 の名前空間規律と一致（系譜: ADR-0007）。

## 4. 非機能要件（NFR）

- **NFR-1（非破壊の絶対性）**: いかなる操作・保存・移行でも Source を書き換えない（例外なし）。
- **NFR-2（可逆性）**: reset to source は決定的に元状態へ戻す。
- **NFR-3（忠実性）**: Source はバイト／意味上、取り込み時と同一を保つ。
- **NFR-4（性能）**: 派生適用のために巨大 Source を無用に複製しない（参照優先）。重い派生生成（軽量化・大画像）は別スレッドへ逃がす（F-PLT-03）。
- **NFR-5（明快さ）**: 元・編集・結果の関係が UI で追える。

## 5. データ・保存への影響

- 本機能は **Source / Derivation / 結果の関係モデル**を定義する。保存は F-CORE-05 の器に、Source を一度・派生を参照つきで持つ（`D4` 名前空間・領域分離を使う）。
- **UI 一時状態は保存に混ぜない**（F-CORE-01 6 章）。選択中の比較表示などは揮発。
- Variant は派生の集合として持ち、Source を共有する（系譜: ADR-0003）。
- 具体的な派生表現（操作列 vs スナップショット派生）や差分保存は実装判断（非目標）。

## 6. 依存機能

- **F-CORE-05**（保存形式・migration）: Source + 派生の永続化・名前空間（相互。F-CORE-05 の器がこのモデルを収める）。
- **F-CORE-01**（ライフサイクル）: 元ファイル非上書き（FR-2）。
- **F-CORE-04**（Undo/Redo）: 派生への編集操作の履歴移動は F-CORE-04 が担う（**F-CORE-04 は本機能に依存**。非破壊モデルが substrate、Undo/Redo が航法）。
- 各エディションの編集（**F-2D-05/06** ラスタ・ベクタ、**F-3D-10/11/12** 軽量化・比較）は本モデルの上に載る。

## 7. 受け入れ条件（Acceptance）

- **AC-1**: 任意の編集列のあとでも、Source が取り込み時とバイト／意味上同一（NFR-1/NFR-3）。
- **AC-2**: reset to source で、未編集の元状態へ正しく戻る（FR-5）。
- **AC-3**: Bake / Flatten が新しい派生を作りつつ、**Source が再編集可能なまま残る**（FR-6）。
- **AC-4**: 前後比較が、2D は Source vs 現派生、3D は pre/post optimize を並べて示す（FR-8）。
- **AC-5**: 1 Source から作った 2 つの Variant が独立し、片方の編集が他方・Source に影響しない（FR-7）。
- **AC-6**: 取り込んだ元ファイルが、いかなる保存・書き出しでもディスク上で不変（FR-2、F-CORE-01 AC-4 / F-CORE-05 AC-6 と整合）。

## 8. 非目標

- Undo/Redo の履歴移動そのもの（F-CORE-04）。
- 具体的な 2D/3D 編集操作の定義（各エディション機能）。
- 保存の物理形式・差分保存方式（F-CORE-05／実装判断）。
- バリアントの一括生成 UI（F-2D-14 バッチ&バリアント）。
