# 素材検査ユーザーガイド

作成日: 2026-07-16
対象: `2D-3-TYPE-PROFILES + 2D-3-INSPECT`
採用契約: `A+B+X`

## 1. 素材検査とは

Editorのプロパティ欄にある「素材検査（読み取り専用）」は、選択中の素材をゲームで使う前に、不足、矛盾、確認した方がよい情報を一覧で示します。

検査結果には、次の内容が表示されます。

- 何が不足または矛盾しているか
- なぜ確認が必要か
- どのように直す、または確認するか
- Editorのどの場所を確認するか
- 文書やテストで参照できる検査コード

検査結果は素材データへ保存されません。検査を開いたことや問題が表示されたことによって、素材、Project、画像、履歴が変更されることもありません。

## 2. 表示の意味

### 必須確認

現在の素材形式を正しく解釈するために必要な情報の不足や矛盾です。例として、存在しない画像・レイヤー・フレーム・パーツ・取り込み元textureへの参照、正式な取り込み元recordからsource以外への参照、タイル設定の欠落、0以下のエフェクト再生時間があります。

「必須確認」と表示されても、この検査機能が保存、autosave、`.casproj`、exportを新たに止めることはありません。既存の構造検証が以前から拒否している不正データは、従来どおり別の仕組みで拒否されます。

### 推奨確認

ゲームで使う際に役立つことが多い情報です。ただし、用途によって不要な場合があります。静止キャラクターにアニメーションを付けない、背景をループさせない、といった設計も許容されます。

表示された直し方を確認し、今回の素材に必要な場合だけ手動で追加してください。

### 情報

将来の検査拡張で、問題ではない補足を表示するために予約している分類です。保存やexportの停止条件には使いません。

## 3. 現在検査する素材種別

最初の実装では、現行の6種だけを検査します。

| 種別 | 必須確認の例 | 推奨確認の例 |
|---|---|---|
| character | 参照切れ、正しくないキャンバスサイズ | body判定、アニメーション、アンカー |
| item | 参照切れ、正しくないキャンバスサイズ | pickup判定、itemタグまたはゲーム属性 |
| background | 参照切れ、正しくないキャンバスサイズ | 背景レイヤー設定、ループ、視差速度 |
| tile | タイル設定、タイルサイズ、キャンバス内の整合 | 分割時の余り、当たり判定タイプ |
| gimmick | ギミック設定 | 移動方法、bodyまたはsensor判定、用途タグ |
| effect | エフェクト設定、正の再生時間 | アニメーション長、ループ、発生位置アンカー |

UI素材とicon素材は、専用の保存形式と移行方法を決める契約まで未対応です。`item`や`effect`として仮登録し、正式対応済みのように扱うことはしません。

## 4. 基本的な使い方

1. Editorで確認する素材を選びます。
2. プロパティ欄の「ゲーム属性」内にある「素材検査（読み取り専用）」を確認します。
3. 各項目の「理由」「直し方・確認方法」「確認場所」を読みます。
4. 必要な項目だけ、案内されたパネルで手動修正します。
5. 修正後、検査結果が自動で更新されたことを確認します。

自動修復ボタンはありません。検査機能がテンプレートを無断適用したり、データを削除したりすることもありません。

## 5. 対応範囲外

この検査は、対象別exportの合否判定ではありません。Unity、Godot、RPG Makerなど、出力先ごとの停止条件は後続の`2D-4-PREFLIGHT`で別に扱います。

次の項目も今回の範囲外です。

- UI / iconの正式なAsset type追加
- schema、version、migrationの変更
- Family / Variant、linked更新
- 自動修復
- 検査結果の保存
- inspectorの重大度を使った保存・export停止
