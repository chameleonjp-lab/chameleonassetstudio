## 1. 5 分で分かる要約（非エンジニア向け）

Chameleon Asset Studio の 3D 機能は、**3D モデルを「作る」道具ではなく、外部（AI や配布素材）で作られた 3D モデルを「ゲームで使える素材へ整える」道具**にする。

できるようになること: 3D ファイル（GLB / glTF）を読み込む → 壊れていないか・重すぎないかを検品する → 原点・向き・大きさを整える → 武器の位置や当たり判定などのゲーム用情報を付ける → 保存していつでも再開できる → 説明書つきの ZIP に書き出してゲームへ持ち込む。

作り方は四段階に分ける。

| 段階 | 一言でいうと | 終わったときの状態 |
|---|---|---|
| 開始前 Gate | 始めてよいかの確認と土台の決定 | 2D を壊さない証拠・使うライブラリ・データの入れ物が決まっている |
| 第一段階 | 最低限の一続きの体験 | GLB を読み込み、検品し、設定し、保存し、書き出せる（行き止まりなし） |
| 第二段階 | 実用に耐える広がり | glTF 一式・モデルの中身の確認・アニメーション再生・持ち込み手順書・スマホ対応 |
| 第三段階 | 高度な検品と軽量化 | 元を壊さずに軽量化した別モデルを作り、前後を比較して選べる |
| 第四段階 | 完成 | 完成条件 12 項目を証拠つきで満たし、人間が完成を宣言する |

守ること: 既存の 2D 機能・保存データを一切壊さない。元の 3D モデルを絶対に上書きしない。生成 AI をアプリ本体に入れない。ライセンス未確認のものを使わない。

2026-07-20 の改訂で追加した内容: VRM（VRoid 等の人型アバター形式）の検出と利用条件表示、VR 向け検品プリセット、骨（スケルトン）の検査と「右手・頭」などの対応付け、テクスチャの修正（2D 編集機能の再利用）、Blender / Unreal への持ち込み手順書、画像から 3D を作る外部ツールとの接続詳細。いずれも「作るのは外部・整えるのは Chameleon」という役割分担は変えていない（詳細: `3D_INTEROP_VRM_VR_AND_CREATION_SPEC.md`）。
